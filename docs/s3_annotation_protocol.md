# S3 cardiologist relabeling protocol

This document specifies how to obtain ground-truth S3 (third heart sound)
labels on top of the public CirCor DigiScope 2022 dataset. It is intended
to be handed to clinicians as the basis for an annotation task.

The protocol assumes the synthetic-S3-augmented detector
(`openstetho_model.train_s3`) has already been trained and is operating as
a candidate generator. The clinicians are not labeling raw recordings from
scratch — they are reviewing **cycle-level candidates** ranked by the model.

## Background

S3 is a low-frequency (30–70 Hz, dominant ~40 Hz), short (40–80 ms),
early-diastolic sound generated by rapid ventricular filling. In adults,
it is associated with elevated LV filling pressure and is a clinical sign
of HFrEF / HFpEF; in children, it is often physiologic. Detection is
clinically reported per cycle ("S3 audible in 3 of 7 cycles").

Public PCG corpora (CirCor 2022, PhysioNet 2016, PASCAL 2011) do **not**
include S3 labels. Without ground truth we cannot train, validate, or
calibrate. This protocol produces the labels.

## Scope

* Source recordings: the publicly-released CirCor DigiScope 2022 training
  set (4 auscultation locations × ~942 pediatric patients).
* Annotation unit: one cardiac cycle, defined as
  `[S1_n, S1_{n+1})` from our heuristic segmenter.
* Label space (per cycle):

      0 — no S3
      1 — S3 present (any plausible early-diastolic low-frequency sound)
      9 — unannotatable (segmenter mislocated S1/S2, signal corrupted)

  Label `9` cycles are excluded from training and test metrics.

## Subset selection

CirCor is pediatric — physiologic S3 is common, which is useful for
recall training but conflates "S3 present" with "S3 pathologic". For the
first annotation pass, sample:

* 200 recordings stratified by Murmur ∈ {Present, Absent}.
* Within each stratum, oversample model-positive candidates: take the
  top-50 recordings ranked by clip-level max sigmoid from a model trained
  on synthetic S3 only. The remaining 50 per stratum are uniformly
  random. This gives the annotators a meaningful fraction of true
  positives without biasing the dataset entirely to model-positive
  decisions.

Total ≈ 200 recordings × ~10 cycles/recording = ~2000 cycles per pass.

## Workflow

1. Run `openstetho_model.train_s3` with synthetic augmentation only,
   producing `runs/s3_synth/best.pt`.
2. Score every CirCor recording with
   `openstetho_model.validate_clips`-style cycle-level inference, saving
   per-cycle sigmoid scores.
3. Build the annotation manifest (subset selection above) and produce a
   small browser-based viewer that for each cycle shows:
   * 2 s of audio centered on the cycle, playback rate selectable.
   * The segmenter's S1/S2 markers as vertical lines.
   * The cycle-level mel-spectrogram with the diastolic window shaded.
   * Two label buttons (0, 1) and an "unannotatable" button (9).
4. Send each cycle to **two independent cardiologists** blind to each
   other's labels and to the model's score.
5. Compute Cohen's kappa between raters on the binary (0/1) subset.
6. Resolve disagreements by a third senior reviewer, producing the gold
   label.

## Inter-rater agreement target

* κ ≥ 0.7 on the binary S3 label is the gate for using the dataset for
  training and test split. κ below this means the protocol is too
  ambiguous; revise the viewer and reconvene before scaling annotation.
* Report κ in the dataset card alongside per-rater confusion matrices.

## Compensation and IRB

* CirCor is already deidentified and publicly released, so this work is
  re-annotation of public data. No human-subjects approval required.
  Confirm exemption status with the local IRB before kickoff.
* Pay per cycle, not per hour, to keep throughput honest. A rough
  starting rate is ~$0.50/cycle with a $X minimum batch fee.

## Training / test split

* Hold out 25 % of patients (not cycles) as the test set, stratified by
  Murmur label.
* Never train on a patient that contributed cycles to the test set,
  including augmentation. Patient-level leakage destroys the metrics
  this protocol exists to produce.

## Deliverables

* `data/s3_circor_labels.csv` — columns:
  `patient_id, location, cycle_no, s1_idx_samples, s2_idx_samples,
  rater_a, rater_b, gold_label, kappa_subset`
* `docs/s3_annotation_kappa.md` — κ, per-rater confusion matrices,
  examples of cycles that drove resolutions.
* `docs/s3_dataset_card.md` — a short dataset card following the
  Mitchell et al. "Model Cards" / "Datasheets for Datasets" format
  adapted for an annotation dataset.

## Out of scope here

* S4 labeling (different sound, different model). Plan a separate
  protocol if/when an S4 head is added.
* Real-EF correlation. EF is an echo measurement; CirCor has no echo.
  See [[ef-detection-plan]] for the path to EF labels.
* Adult corpora. If a paired adult PCG + echo cohort becomes available,
  apply the same workflow; pediatric-vs-adult drift will require
  separate calibration.
